# Token Forest — 隐私说明 🔒

[English](PRIVACY.md) · [简体中文](PRIVACY.zh-CN.md)

Token Forest 诞生的初衷，只是想用一种治愈的方式*看见*我们自己的 AI 用量——不是为了收集任何人的数据。这一页用大白话讲清楚：应用到底碰了什么。

## 唯一的铁律

**你机器上的东西，只留在你机器上。**

## Token Forest 会读什么

Claude Code 和 Codex 本来就会在你电脑上留下用量日志（它们自己的统计功能也靠这个）。Token Forest 只读这些本地文件里的 **token 数量**：

- 输入 token、输出 token、缓存读 token、缓存写 token
- 每条记录的模型名和时间戳（用于各模型统计和每日曲线）

就这些，没别的。

## Token Forest 绝不会读什么

- ❌ 你的源代码
- ❌ 你的提示词或 AI 的回复
- ❌ 你的文件、项目，或磁盘上任何其他东西

成长逻辑需要的是一个数字，不是你的作品。它拿到的永远只是那个数字。

## Token Forest 会存什么

在应用旁边存几个本地小文件：树的成长状态、你的设置（窗口位置、语言等）、以及聚合后的每日用量总数。全部在本地。删掉它们，应用就重新开始。

## 联网

**默认完全不联网。** 没有遥测、没有分析、没有更新探测、没有价格拉取（成本估算用的是内置的离线价目表）。

**唯一**的联网功能是**排行榜**，且严格自愿：

| | |
| --- | --- |
| 会上传什么 | 你自选的昵称、地区、当前树种、树的 token 总量、成长阶段 |
| 如何标识你 | 一个匿名账户——无邮箱、无姓名、无机器信息 |
| 如何退出 | 关闭排行榜会从服务器**删除你的记录** |

## 如何验证这些承诺

我们很清楚"相信我们"这句话本身不够分量。源代码目前是私有的，但我们在考虑开放核心的用量读取组件，让这些承诺可以被独立审计。在那之前：断网状态下应用照常运行——你可以亲自试试。
